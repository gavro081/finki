explanation of what each container does:
the git-sync container periodically pulls content from a remote git repository and writes it into a shared emptyDir volume mounted at /html.
the ctr-web container is a web server that serves files from that same /html volume. neither container depends on the other's image or internal logic, they are decoupled and communicate only through the shared volume.
the shared volume is the key to this sidecar pattern because it allows two containers with completely different responsibilities to collaborate without any direct coupling, the sidecar enriches or supplies data that the main container consumes, which is the core idea behind the pattern.
